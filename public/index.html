a permissionless hacker news for anons who build peer-to-peer.

pear bare hypercore hyperswarm p2p-hiverelay

01.who we are

we're the people building peer-to-peer software because we think the internet should work without asking permission.

no servers. no sign-ups. no account recovery. no companies to shut down. no terms of service. no one to complain to. just keys, protocols, and code.

if centralized services are fragile, we work on the alternative. if your data lives in someone else's database, you don't own it. if your identity requires their approval, you don't have one. this isn't a rant, it's a building project.

anonymity here isn't pretending. it's the default. everyone on this network is a 32-byte public key. whatever name you go by is the one you pick.

02.the message board

p2pbuilders is a hacker news clone that runs on no servers. every user runs their own node. posts sync peer-to-peer via hyperswarm. every post is signed by its author's hypercore. reputation-weighted voting, hashcash-gated submissions, no central moderator.

it lives in your terminal. no web app. no phone app. no electron bloat. you type pear run pear://<key> and you're on it.

03.join

takes two minutes. you'll need node.js installed.

  1. install pear — the p2p runtime holepunch ships.

    npm install -g pear
pear

    the second command bootstraps the real runtime. takes 30s on first run.

  2. add pear to your path. one-time setup.

    echo 'export PATH="$HOME/Library/Application Support/pear/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
which pear
# should print: /Users/you/Library/Application Support/pear/bin/pear

  3. run the board.

    pear run pear://gopfpwat99tcuaakasfnftrds3j6t7srdmi3qidbhm9xeizt1a5y

    type help at the › prompt. submit a post to say hi.
